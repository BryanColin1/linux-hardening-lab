# Linux Hardening Lab

## Overview

This project demonstrates basic Linux system hardening in a controlled lab environment. The objective is to review a Linux system baseline, identify common security weaknesses, apply hardening steps, and document before-and-after security improvements.

## Objectives

- Perform a basic Linux security baseline review
- Review users, sudo access, services, open ports, and firewall status
- Apply basic Linux hardening practices
- Document hardening steps and verification results
- Create a practical hardening checklist

## Lab Environment

| Component | Description |
|---|---|
| Target System | Ubuntu or Kali Linux VM |
| Platform | VirtualBox |
| Network | Local lab environment |
| Documentation | Markdown and GitHub |

## Planned Hardening Areas

| Area | Focus |
|---|---|
| User Accounts | Review users, groups, and sudo privileges |
| SSH Security | Review and harden SSH configuration |
| Firewall | Enable and configure UFW |
| Services | Identify and disable unnecessary services |
| Updates | Apply package updates and patch management |
| Logging | Review basic authentication and system logs |

## Skills Demonstrated

- Linux system administration
- Security baseline review
- User and privilege review
- Firewall configuration
- Service hardening
- SSH hardening
- Security documentation
- Remediation planning

## Disclaimer

This project is performed in a controlled local lab environment for educational and defensive security purposes only. No production systems or third-party systems are tested.
