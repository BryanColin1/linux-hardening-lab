# Lab Scope

## Purpose

This lab demonstrates basic Linux hardening practices in a controlled virtual machine environment. The goal is to review baseline security settings, apply hardening steps, and document improvements.

## Environment

The lab is performed on a Linux virtual machine owned and controlled by me.

## In Scope

- User account review
- Sudo privilege review
- SSH configuration review
- Firewall status review
- Open port review
- Running service review
- Package update review
- Basic log review
- Hardening documentation

## Out of Scope

- Production systems
- Third-party systems
- Unauthorized testing
- Malware deployment
- Exploit development
- Destructive system changes
- Real user credential collection

## Rules of Engagement

- Only test systems owned and controlled in the lab
- Take screenshots only after confirming no sensitive information is visible
- Do not upload passwords, private keys, or tokens
- Document changes clearly before and after hardening
- Avoid destructive configuration changes unless they are understood and reversible
